# 第五部分：自动求导与动态计算图

## 第14章：动态计算图构建

### 14.1 计算图概念

计算图是自动求导系统的核心数据结构，用于记录计算过程和依赖关系。

#### 14.1.1 节点：Tensor

**计算图节点架构：**

```
┌─────────────────────────────────────┐
│  计算图节点（Tensor）                │
├─────────────────────────────────────┤
│                                      │
│  data_ (shared_ptr<Mat>)            │
│    └─> 节点数据                      │
│                                      │
│  grad_ (shared_ptr<Mat>)            │
│    └─> 节点梯度                      │
│                                      │
│  creator_ (FunctionPtr)             │
│    └─> 创建者 Operator               │
│         └─> 建立与边的连接           │
│                                      │
│  generation_ (int)                  │
│    └─> 拓扑排序用                    │
│                                      │
└─────────────────────────────────────┘
```

**节点连接关系：**

```
计算图节点关系：
  Tensor (节点)
    │
    ├─> data_ ──> 实际数据
    ├─> grad_ ──> 梯度数据
    └─> creator_ ──> Operator (边)
          │
          └─> 连接到前一个节点
```

#### 14.1.2 边：Operator

**计算图边架构：**

```
┌─────────────────────────────────────┐
│  计算图边（Operator）                │
├─────────────────────────────────────┤
│                                      │
│  inputs_ (vector<Tensor>)           │
│    └─> 输入节点                      │
│                                      │
│  outputs_ (weak_ptr<TensorImpl>)     │
│    └─> 输出节点（避免循环引用）       │
│                                      │
│  generation_ (int)                  │
│    └─> 拓扑排序用                    │
│                                      │
│  forward() / backward()             │
│    └─> 前向/反向传播                 │
│                                      │
└─────────────────────────────────────┘
```

**边连接关系：**

```
计算图边连接：
  Operator (边)
    │
    ├─> inputs_ ──> [Tensor1, Tensor2]  (输入节点)
    └─> outputs_ ──> [Tensor3]          (输出节点)
          │
          └─> 连接输入和输出节点
```

#### 14.1.3 前向传播时自动构建

**计算图构建架构：**

```
前向传播时自动构建：
  ┌─────────────────────────────────────┐
  │  用户代码                            │
  │    auto y = x * 2;                  │
  └─────────────────────────────────────┘
              │
              ▼
  ┌─────────────────────────────────────┐
  │  Operator::operator()               │
  │    1. forward() 计算输出             │
  │    2. 设置 creator_ (建立连接)      │
  │    3. setup_computation_graph()     │
  └─────────────────────────────────────┘
              │
              ▼
  ┌─────────────────────────────────────┐
  │  计算图连接建立                      │
  │    x ──[Mul]──> y                   │
  └─────────────────────────────────────┘
```

**构建流程：**

```
计算图构建流程：
  1. Operator 调用 forward()
     └─> 计算输出 Tensor
  
  2. 设置 creator_
     └─> output.creator_ = Operator
  
  3. 保存输入输出信息
     └─> Operator.inputs_ = [x]
     └─> Operator.outputs_ = [y]
  
  4. 设置 generation_
     └─> y.generation_ = x.generation_ + 1
```

#### 14.1.4 动态图的优势

**动态图优势架构：**

```
┌─────────────────────────────────────┐
│  动态图优势                          │
├─────────────────────────────────────┤
│                                      │
│  ✅ 灵活性                           │
│     计算图结构可以动态变化            │
│     支持条件分支、循环                │
│                                      │
│  ✅ 易用性                           │
│     用户代码直观                     │
│     不需要显式构建图                 │
│                                      │
│  ✅ 易调试                           │
│     可以打印中间结果                 │
│     可以检查计算图结构                │
│                                      │
└─────────────────────────────────────┘
```

**动态图示例：**

```
动态图结构变化：
  ┌─────────────────────────────────────┐
  │  if (condition)                     │
  │    y = x * 2;  ──> 路径1            │
  │  else                               │
  │    y = x + 1;  ──> 路径2            │
  └─────────────────────────────────────┘
  
  根据条件动态构建不同的计算图
```

### 14.2 计算图连接机制

#### 14.2.1 creator_ 字段的作用

**creator_ 连接架构：**

```
计算图连接机制：
  ┌─────────────────────────────────────┐
  │  Tensor (节点)                      │
  │    creator_ ──> Operator (边)       │
  │      │                              │
  │      └─> 建立从节点到边的连接        │
  └─────────────────────────────────────┘
              │
              ▼
  ┌─────────────────────────────────────┐
  │  反向传播时回溯                      │
  │    backward()                       │
  │      └─> 从 creator_ 开始           │
  │            └─> 回溯计算图            │
  └─────────────────────────────────────┘
```

**creator_ 作用：**

```
┌─────────────────────────────────────┐
│  creator_ 作用                      │
├─────────────────────────────────────┤
│                                      │
│  ✅ 记录创建者 Operator               │
│                                      │
│  ✅ 建立节点到边的连接                │
│                                      │
│  ✅ 反向传播时回溯计算图              │
│                                      │
└─────────────────────────────────────┘
```

#### 14.2.2 generation_ 的作用（拓扑排序）

**generation_ 拓扑排序架构：**

```
计算图代数设置：
  输入: generation_ = 0
    ↓
  Add: generation_ = 1
    ↓
  Mul: generation_ = 2
    ↓
  输出: generation_ = 3

反向传播顺序：
  3 → 2 → 1 → 0  (从大到小)
```

**拓扑排序流程：**

```
┌─────────────────────────────────────┐
│  拓扑排序流程                        │
├─────────────────────────────────────┤
│                                      │
│  1. 按 generation_ 排序              │
│     └─> 从大到小                     │
│                                      │
│  2. 从后往前处理                     │
│     └─> generation_ 大的先处理       │
│                                      │
│  3. 确保梯度计算顺序                 │
│     └─> 先计算输出梯度               │
│         └─> 再计算输入梯度           │
│                                      │
└─────────────────────────────────────┘
```

#### 14.2.3 计算图信息设置

**setup_computation_graph() 方法：**
```cpp
void Operator::setup_computation_graph(const std::vector<Tensor> &inputs,
                                       const std::vector<Tensor> &outputs) {
    // 保存输入
    inputs_ = inputs;
    
    // 保存输出（使用 weak_ptr 避免循环引用）
    outputs_.clear();
    for (const auto &output : outputs) {
        outputs_.push_back(output.impl_);  // 存储 weak_ptr
    }
    
    // 设置 generation_
    if (!inputs.empty() && inputs[0].impl_) {
        generation_ = inputs[0].impl_->generation_ + 1;
    } else {
        generation_ = 0;
    }
}
```

**关键点：**
- **inputs_**：保存输入 Tensor（用于 backward()）
- **outputs_**：保存输出 TensorImpl 的 weak_ptr（避免循环引用）
- **generation_**：设置为输入的最大 generation_ + 1

### 14.3 计算图结构

#### 14.3.1 单输入单输出

**结构：**
```
x → [ReLU] → y
```

**特点：**
- 最简单的计算图结构
- 一个输入，一个输出
- 反向传播路径唯一

**示例：**
```cpp
auto x = Tensor::randn({2, 3});
auto y = relu(x);  // 单输入单输出
y.backward();      // 反向传播
```

#### 14.3.2 多输入多输出

**结构：**
```
x0 ──┐
     ├→ [Add] → y
x1 ──┘
```

**特点：**
- 多个输入，一个输出
- 反向传播时计算多个梯度
- 梯度需要广播（如果形状不同）

**示例：**
```cpp
auto x0 = Tensor::randn({2, 3});
auto x1 = Tensor::randn({2, 3});
auto y = x0 + x1;  // 多输入单输出
y.backward();      // 计算 gx0 和 gx1
```

#### 14.3.3 分支与合并

**结构：**
```
     ┌→ [Mul] → y1
x ───┤
     └→ [Add] → y2
```

**特点：**
- 一个输入，多个输出
- 反向传播时梯度从多个路径传播
- 需要梯度累积

**示例：**
```cpp
auto x = Tensor::randn({2, 3});
auto y1 = x * 2;
auto y2 = x + 1;
// x 被两个算子使用，反向传播时需要累积梯度
```

#### 14.3.4 循环结构

**结构：**
```
x → [Op1] → y1 → [Op2] → y2 → [Op3] → y3
                                      ↓
                                    [Op4] → y4
```

**特点：**
- 链式结构，多个算子串联
- 反向传播时按相反顺序处理
- generation_ 递增

**示例：**
```cpp
auto x = Tensor::randn({2, 3});
auto y1 = x * 2;
auto y2 = y1 + 1;
auto y3 = relu(y2);
y3.backward();  // 反向传播：y3 → y2 → y1 → x
```

## 第15章：反向传播实现

### 15.1 backward() 方法架构

#### 15.1.1 入口：TensorImpl::backward()

**调用方式：**
```cpp
auto x = Tensor::randn({2, 3});
auto y = x * 2;
y.backward();  // 用户调用
```

**入口方法：**
```cpp
void TensorImpl::backward() {
    // 1. 检查是否启用反向传播
    if (!Config::enable_backprop) {
        return;
    }
    
    // 2. 初始化梯度
    if (!grad_) {
        grad_ = create_ones_tensor();  // 全1张量
    }
    
    // 3. 拓扑排序和梯度计算
    // ...
}
```

#### 15.1.2 梯度初始化（全1张量）

**初始化规则：**
- 输出张量的梯度初始化为全1
- 梯度形状与数据形状相同
- 梯度类型与数据类型相同

**实现：**
```cpp
if (!grad_) {
    auto data_device = data_->device();
    TensorOptions options = TensorOptions(data_->dtype()).device(data_device);
    grad_ = std::shared_ptr<Mat>(Mat_t::from_scalar(1, data_->shape(), options));
}
```

**为什么是全1？**
- 输出张量对自身的梯度是1（∂y/∂y = 1）
- 作为反向传播的起点
- 后续通过链式法则传播

#### 15.1.3 拓扑排序算法（generation_ 排序）

**排序目的：**
- 确保梯度计算的正确顺序
- 从输出到输入，按依赖关系处理
- 避免重复计算

**实现：**
```cpp
auto funcs = std::list<FunctionPtr>();
auto func_set = std::set<FunctionPtr>();

auto add_func = [&funcs, &func_set](const FunctionPtr &f) {
    if (f && func_set.find(f) == func_set.end()) {
        funcs.push_back(f);
        func_set.insert(f);
        // 按 generation_ 排序
        funcs.sort([](const FunctionPtr &lhs, const FunctionPtr &rhs) {
            return lhs->generation_ < rhs->generation_;
        });
    }
};

// 从当前 Tensor 的 creator_ 开始
add_func(this->creator_);
```

**处理顺序：**
```cpp
while (!funcs.empty()) {
    auto f = funcs.back();  // 取 generation_ 最大的
    funcs.pop_back();
    
    // 处理这个 Operator
    // ...
    
    // 递归处理输入的 creator_
    for (auto &input : f->inputs_) {
        if (input.impl_->creator_) {
            add_func(input.impl_->creator_);
        }
    }
}
```

### 15.2 拓扑排序实现

#### 15.2.1 使用 list 和 set 维护待处理队列

**数据结构：**
```cpp
std::list<FunctionPtr> funcs;      // 待处理的 Operator 队列
std::set<FunctionPtr> func_set;    // 已添加的 Operator（去重用）
```

**为什么使用 list？**
- 支持排序操作
- 支持高效的插入和删除
- 保持顺序

**为什么使用 set？**
- 快速查找（O(log n)）
- 去重，避免重复处理同一个 Operator
- 防止循环依赖导致的无限循环

#### 15.2.2 generation_ 的作用

**排序规则：**
```cpp
funcs.sort([](const FunctionPtr &lhs, const FunctionPtr &rhs) {
    return lhs->generation_ < rhs->generation_;  // 从小到大排序
});
```

**处理顺序：**
- 从小到大排序后，从后往前处理
- generation_ 大的先处理（输出端）
- generation_ 小的后处理（输入端）

**示例：**
```
generation_ = 3: 输出端，先处理
generation_ = 2: 中间层
generation_ = 1: 中间层
generation_ = 0: 输入端，后处理
```

#### 15.2.3 去重机制（避免重复处理）

**问题：**
- 一个 Operator 可能被多个路径引用
- 需要避免重复计算梯度

**解决方案：**
```cpp
auto add_func = [&funcs, &func_set](const FunctionPtr &f) {
    if (f && func_set.find(f) == func_set.end()) {  // 检查是否已添加
        funcs.push_back(f);
        func_set.insert(f);  // 标记为已添加
        // 排序...
    }
};
```

**去重效果：**
- 同一个 Operator 只会被处理一次
- 但梯度会正确累积（在梯度累积阶段）

### 15.3 梯度计算流程

#### 15.3.1 完整流程

```cpp
void TensorImpl::backward() {
    // 1. 初始化梯度（全1）
    if (!grad_) {
        grad_ = create_ones_tensor();
    }
    
    // 2. 初始化待处理队列
    auto funcs = std::list<FunctionPtr>();
    auto func_set = std::set<FunctionPtr>();
    add_func(this->creator_);
    
    // 3. 遍历计算图
    while (!funcs.empty()) {
        auto f = funcs.back();
        funcs.pop_back();
        
        // 4. 收集输出梯度（weak_ptr → shared_ptr）
        std::vector<Tensor> gys;
        for (const auto &weak_output : f->outputs_) {
            auto output_impl = weak_output.lock();
            if (output_impl) {
                Tensor output_tensor(output_impl);
                gys.push_back(output_tensor.grad());
            }
        }
        
        // 5. 调用 Operator::backward(gys)
        auto gxs = f->backward(gys);
        
        // 6. 梯度累积
        for (size_t i = 0; i < gxs.size(); i++) {
            auto x = f->inputs_[i];
            auto gx = gxs[i];
            
            if (!x.impl_->grad_) {
                // 梯度为空，直接共享
                x.impl_->grad_ = gx.impl_->data_;
            } else {
                // 梯度不为空，原地累加
                x.impl_->grad_->add_inplace(*gx.impl_->data_);
            }
            
            // 7. 递归处理输入的 creator_
            if (x.impl_->creator_) {
                add_func(x.impl_->creator_);
            }
        }
    }
    
    // 8. 计算图清理
    // ...
}
```

#### 15.3.2 从 outputs_ 收集梯度（weak_ptr → shared_ptr）

**问题：**
- outputs_ 使用 weak_ptr，可能失效
- 需要转换为 shared_ptr 才能使用

**实现：**
```cpp
std::vector<Tensor> gys;
std::vector<TensorImplPtr> valid_outputs;  // 临时保存，确保生命周期

for (const auto &weak_output : f->outputs_) {
    auto output_impl = weak_output.lock();  // 转换为 shared_ptr
    if (!output_impl) {
        continue;  // 已失效，跳过
    }
    
    // 临时保存，确保在 backward() 期间有效
    valid_outputs.push_back(output_impl);
    
    // 获取梯度
    Tensor output_tensor(output_impl);
    gys.push_back(output_tensor.grad());
}
```

**关键点：**
- 使用 `valid_outputs` 临时保存 shared_ptr
- 确保在 backward() 期间 Tensor 有效
- backward() 结束后自动释放

#### 15.3.3 调用 Operator::backward(gys)

**调用：**
```cpp
auto gxs = f->backward(gys);
```

**验证：**
```cpp
if (gxs.size() != f->inputs_.size()) {
    THROW_RUNTIME_ERROR("backward error!, gxs size {} inputs size {}", 
                        gxs.size(), f->inputs_.size());
}
```

**返回：**
- gxs：输入梯度向量
- 数量必须与输入数量相同

#### 15.3.4 梯度累积

**两种情况：**

1. **梯度为空：直接共享**
   ```cpp
   if (!x.impl_->grad_) {
       x.impl_->grad_ = gx.impl_->data_;  // 直接共享
   }
   ```

2. **梯度不为空：原地累加**
   ```cpp
   else {
       x.impl_->grad_->add_inplace(*gx.impl_->data_);  // 原地累加
   }
   ```

**为什么需要累积？**
- 一个 Tensor 可能被多个 Operator 使用
- 每个路径都会产生梯度
- 需要将所有路径的梯度累加

#### 15.3.5 递归处理输入 tensor 的 creator_

**递归：**
```cpp
for (size_t i = 0; i < gxs.size(); i++) {
    auto x = f->inputs_[i];
    // ...
    
    if (x.impl_->creator_) {
        add_func(x.impl_->creator_);  // 递归添加到队列
    }
}
```

**效果：**
- 从输出端向输入端传播
- 自动处理整个计算图
- 不需要用户手动指定路径

### 15.4 梯度累积设计

#### 15.4.1 直接共享 vs 原地累加

**直接共享（grad_ 为空）：**
```cpp
if (!x.impl_->grad_) {
    x.impl_->grad_ = gx.impl_->data_;  // 直接共享，避免拷贝
}
```

**优势：**
- 避免内存分配
- 避免数据拷贝
- 性能最优

**原地累加（grad_ 不为空）：**
```cpp
else {
    x.impl_->grad_->add_inplace(*gx.impl_->data_);  // 原地累加
}
```

**优势：**
- 不创建新的 Mat 对象
- 直接修改现有数据
- 内存效率高

#### 15.4.2 add_inplace 优化（避免内存分配）

**实现：**
```cpp
void OriginMat::add_inplace(const Mat& other) {
    const OriginMat& other_mat = static_cast<const OriginMat&>(other);
    
    // 原地累加，不创建新对象
    for (size_t i = 0; i < elements(); ++i) {
        data_[i] += other_mat.data_[i];
    }
}
```

**优势：**
- 不分配新内存
- 不创建新对象
- 直接修改现有数据

#### 15.4.3 多路径梯度处理

**场景：**
```
     ┌→ [Mul] ──┐
x ───┤          ├→ [Add] → y
     └→ [Add] ──┘
```

**处理：**
1. 第一条路径：x → Mul → Add → y，计算梯度 gx1
2. 第二条路径：x → Add → Add → y，计算梯度 gx2
3. 累积：gx = gx1 + gx2

**实现：**
- 第一次：grad_ 为空，直接赋值 gx1
- 第二次：grad_ 不为空，累加 gx2
- 结果：grad_ = gx1 + gx2

### 15.5 计算图清理机制

#### 15.5.1 backward() 完成后的自动清理

**清理目的：**
- 释放不再需要的内存
- 断开循环引用
- 减少内存占用

**清理时机：**
- backward() 完成后自动清理
- 不需要用户手动清理

#### 15.5.2 inputs_ 清理（减少内存占用）

**清理：**
```cpp
// 清理 Operator 的 inputs_
for (const auto &f : processed_ops) {
    if (f) {
        f->inputs_.clear();  // 清理输入，减少内存占用
    }
}
```

**为什么可以清理？**
- inputs_ 是值语义的 Tensor
- 清理后不影响其他引用
- backward() 完成后不再需要

#### 15.5.3 grad_ 和 creator_ 清理（断开循环引用）

**清理策略：**
```cpp
for (const auto &tensor_impl : processed_tensors) {
    // 清理 creator_，断开循环引用
    tensor_impl->creator_ = nullptr;
    tensor_impl->generation_ = 0;
    
    // 清理中间 tensor 的 grad_
    if (!is_input_tensor && !is_output_tensor) {
        tensor_impl->grad_ = nullptr;  // 释放梯度内存
    }
}
```

**清理规则：**
- **输入 tensor**：保留 grad_（用户可能需要），清理 creator_
- **输出 tensor**：保留 grad_（用户可能需要），清理 creator_
- **中间 tensor**：清理 grad_ 和 creator_（不再需要）

#### 15.5.4 内存释放策略

**策略：**
1. **立即清理**：backward() 完成后立即清理
2. **选择性清理**：只清理不再需要的部分
3. **保留用户数据**：输入输出 tensor 的 grad_ 保留

**效果：**
- 减少内存占用
- 断开循环引用
- 不影响用户使用

## 第16章：自动求导优化

### 16.1 内存优化

#### 16.1.1 weak_ptr 避免循环引用

**问题：**
- Operator → outputs_ → Tensor → creator_ → Operator
- 使用 shared_ptr 会导致循环引用

**解决方案：**
```cpp
class Operator {
    std::vector<std::weak_ptr<TensorImpl>> outputs_;  // 使用 weak_ptr
};
```

**优势：**
- 不持有强引用，打破循环
- 自动检测对象是否已释放
- 不影响对象的生命周期

**使用：**
```cpp
// forward() 时：保存 weak_ptr
outputs_.push_back(output.impl_);

// backward() 时：转换为 shared_ptr
auto output_impl = weak_output.lock();
if (output_impl) {
    // 使用 output_impl
}
```

#### 16.1.2 backward() 后自动清理计算图

**清理内容：**
1. **inputs_ 清理**：释放 Operator 的输入 Tensor
2. **creator_ 清理**：断开 Tensor 与 Operator 的连接
3. **中间 grad_ 清理**：释放中间 Tensor 的梯度

**实现：**
```cpp
// backward() 完成后
// 1. 清理 inputs_
for (const auto &f : processed_ops) {
    f->inputs_.clear();
}

// 2. 清理 creator_ 和中间 grad_
for (const auto &tensor_impl : processed_tensors) {
    tensor_impl->creator_ = nullptr;
    if (!is_input_tensor && !is_output_tensor) {
        tensor_impl->grad_ = nullptr;
    }
}
```

**效果：**
- 自动释放内存
- 断开循环引用
- 减少内存占用

#### 16.1.3 临时 shared_ptr 的生命周期管理

**问题：**
- backward() 时需要将 weak_ptr 转换为 shared_ptr
- 需要确保在 backward() 期间对象有效

**解决方案：**
```cpp
std::vector<TensorImplPtr> valid_outputs;  // 临时保存

for (const auto &weak_output : f->outputs_) {
    auto output_impl = weak_output.lock();
    if (output_impl) {
        valid_outputs.push_back(output_impl);  // 临时保存
        // 使用 output_impl
    }
}
// valid_outputs 在作用域结束时自动释放
```

**生命周期：**
- 在 backward() 期间保持强引用
- backward() 结束后自动释放
- 不影响对象的实际生命周期

### 16.2 性能优化

#### 16.2.1 梯度检查点（checkpoint）

**概念：**
- 在计算图中设置检查点
- 只保存检查点的中间结果
- 反向传播时重新计算非检查点部分

**优势：**
- 减少内存使用
- 以计算时间换取内存空间
- 适合大模型训练

**实现（未来可能支持）：**
```cpp
// 设置检查点
auto checkpoint = tensor.checkpoint();

// 反向传播时
// 如果遇到检查点，使用保存的结果
// 否则重新计算
```

#### 16.2.2 延迟计算

**概念：**
- 梯度计算延迟到真正需要时
- 不立即计算所有梯度
- 按需计算

**优势：**
- 减少不必要的计算
- 提高效率
- 节省资源

**当前实现：**
- 梯度只在 backward() 时计算
- 不计算不需要的梯度
- 通过 Config::enable_backprop 控制

#### 16.2.3 梯度累积优化

**优化策略：**
1. **直接共享**：第一次梯度直接共享，避免拷贝
2. **原地累加**：后续梯度原地累加，避免分配新内存
3. **批量处理**：批量处理多个梯度

**实现：**
```cpp
if (!x.impl_->grad_) {
    // 第一次：直接共享
    x.impl_->grad_ = gx.impl_->data_;
} else {
    // 后续：原地累加
    x.impl_->grad_->add_inplace(*gx.impl_->data_);
}
```

### 16.3 禁用梯度计算

#### 16.3.1 Config::enable_backprop

**全局配置：**
```cpp
namespace Config {
    extern bool enable_backprop;  // 默认 true
}
```

**使用：**
```cpp
// 禁用梯度计算
Config::enable_backprop = false;

// 前向传播（不构建计算图）
auto y = x * 2;

// 恢复
Config::enable_backprop = true;
```

**检查：**
```cpp
void TensorImpl::backward() {
    if (!Config::enable_backprop) {
        return;  // 直接返回，不计算梯度
    }
    // ...
}
```

#### 16.3.2 推理模式优化

**no_grad() 上下文管理器：**
```cpp
{
    auto guard = no_grad();  // 禁用梯度计算
    auto y = model(x);       // 推理，不构建计算图
    // guard 析构时自动恢复
}
```

**实现：**
```cpp
class NoGradGuard {
private:
    bool old_value_;
    
public:
    NoGradGuard() : old_value_(Config::enable_backprop) {
        Config::enable_backprop = false;  // 禁用
    }
    
    ~NoGradGuard() {
        Config::enable_backprop = old_value_;  // 恢复
    }
};

NoGradGuard no_grad() {
    return NoGradGuard();
}
```

**优势：**
- RAII 模式，自动管理
- 支持嵌套使用
- 异常安全

#### 16.3.3 内存节省策略

**策略 1：禁用梯度计算**
```cpp
// 推理时禁用梯度计算
{
    auto guard = no_grad();
    auto output = model(input);  // 不构建计算图，节省内存
}
```

**策略 2：及时清理**
```cpp
// backward() 后自动清理
loss.backward();
// 计算图已自动清理，内存已释放
```

**策略 3：选择性保留**
```cpp
// 只保留需要的梯度
loss.backward();
auto param_grad = param.grad();  // 获取梯度
param.clear_grad();               // 清除不需要的梯度
```

**效果：**
- 推理时节省大量内存
- 训练时及时释放不需要的内存
- 提高内存使用效率
