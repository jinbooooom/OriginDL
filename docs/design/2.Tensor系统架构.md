# 第二部分：Tensor 系统架构

## 第3章：Tensor 分层架构设计

### 3.1 Tensor 系统整体架构

Tensor 系统采用四层架构设计，从用户接口到数据存储，每一层都有明确的职责和清晰的边界。

#### 3.1.1 四层架构概览

### 3.2 Tensor 层：值语义包装

Tensor 层是用户直接接触的接口层，采用值语义设计，让 Tensor 对象可以像普通值一样传递和复制，提供直观易用的 API。

#### 3.2.1 值语义设计原理

**设计原理：**

值语义是指对象的行为像普通值一样。在 OriginDL 中，Tensor 采用**值语义 + 底层共享**的设计：

- **用户视角**：值语义，拷贝时创建独立副本
- **底层实现**：浅拷贝，共享底层 TensorImpl

**类设计：**

```cpp
class Tensor {
private:
    TensorImplPtr impl_;  // 唯一的成员：shared_ptr<TensorImpl>
    
public:
    // 拷贝构造函数 - 浅拷贝，共享实现
    Tensor(const Tensor &other) : impl_(other.impl_) {}
    
    // 赋值运算符 - 浅拷贝
    Tensor &operator=(const Tensor &other) {
        impl_ = other.impl_;
        return *this;
    }
    
    // 运算符重载（返回新对象）
    Tensor operator+(const Tensor& other) const;
    Tensor operator*(const Tensor& other) const;
};
```

**内存模型对比：**

```
值语义（用户视角）：
  Tensor x ──┐
             ├─> 两个独立的 Tensor 对象
  Tensor y ──┘

底层实现（共享）：
  Tensor x
    └─> impl_ (shared_ptr) ──┐
                              ├─> 同一个 TensorImpl
  Tensor y                    │
    └─> impl_ (shared_ptr) ──┘
```

**为什么选择值语义？**

1. **易用性**：符合直觉，降低学习成本
   ```cpp
   // 值语义：直观
   auto z = x + y;
   
   // 引用语义：需要理解所有权
   auto z = x.add(y);  // x 会被修改吗？
   ```

2. **安全性**：避免意外的副作用
   - 值语义下，`y = x` 不会修改 `x`
   - 引用语义下，需要小心处理所有权

3. **与 PyTorch 一致**：降低迁移成本

#### 3.2.2 浅拷贝机制（shared_ptr 共享 TensorImpl）

**核心机制：**

- Tensor 对象只包含一个 `shared_ptr<TensorImpl>` 成员
- 拷贝 Tensor 时，只拷贝智能指针，共享底层的 TensorImpl
- 多个 Tensor 对象可以共享同一个 TensorImpl

**内存结构图：**

```
拷贝前：
┌──────────┐
│ Tensor x │
│──────────│
│ impl_ ───┼──> [TensorImpl] (ref_count=1)
└──────────┘

拷贝后：
┌──────────┐
│ Tensor x │
│──────────│
│ impl_ ───┼──┐
└──────────┘  │
              │ shared_ptr
              │ (ref_count=2)
┌──────────┐  │
│ Tensor y │  │
│──────────│  │
│ impl_ ───┼──┘
└──────────┘
              │
              ▼
        [TensorImpl]
        (共享同一个对象)
```

**引用计数生命周期：**

```
1. 创建 Tensor x:
   TensorImpl 引用计数 = 1

2. 拷贝 Tensor y = x:
   TensorImpl 引用计数 = 2

3. x 超出作用域:
   TensorImpl 引用计数 = 1

4. y 超出作用域:
   TensorImpl 引用计数 = 0
   └─> 自动释放 TensorImpl
```

**优势：**
- **性能**：避免不必要的深拷贝
- **内存效率**：多个 Tensor 共享底层数据
- **语义清晰**：值语义，但底层共享

#### 3.1.3 用户接口设计原则

**接口设计架构：**

```
┌─────────────────────────────────────┐
│  Tensor 接口设计原则                 │
├─────────────────────────────────────┤
│                                      │
│  1. 隐藏实现细节                     │
│     └─> 用户不需要知道 TensorImpl   │
│         └─> 所有操作通过 Tensor 接口│
│                                      │
│  2. 不可变接口                       │
│     └─> 大部分操作返回新 Tensor     │
│         └─> reshape(), transpose()  │
│                                      │
│  3. 链式调用支持                     │
│     └─> x.reshape().transpose()     │
│                                      │
│  4. 类型安全                         │
│     └─> 编译时检查                   │
│                                      │
└─────────────────────────────────────┘
```

#### 3.1.4 运算符重载策略

**运算符重载架构：**

```
用户代码层：
  auto z = x + y;  ← 值语义，直观

架构设计：
  ┌─────────────────────────────────────┐
  │  Tensor::operator+()                │
  │    └─> 创建 Add 算子                 │
  │         └─> 调用算子层              │
  │              └─> 返回新 Tensor      │
  └─────────────────────────────────────┘

设计特点：
  ✅ 所有运算符返回新 Tensor（不可变）
  ✅ 通过算子层实现，不直接操作底层
  ✅ 支持自动求导
```

### 3.2 TensorImpl 层：核心实现

TensorImpl 是 Tensor 的核心实现层，负责管理张量的数据、梯度和计算图信息。

#### 3.2.1 职责定位

TensorImpl 承担三个核心职责：

1. **数据管理**
   - 管理张量的数据（data_）
   - 管理梯度（grad_）
   - 通过 Mat 抽象层访问底层数据

2. **计算图管理**
   - 记录创建者（creator_）
   - 记录生成代数（generation_）
   - 支持反向传播

3. **生命周期管理**
   - 使用智能指针管理内存
   - 支持共享和拷贝

#### 3.2.2 数据成员设计

**TensorImpl 数据成员架构：**

```
┌─────────────────────────────────────┐
│         TensorImpl                  │
├─────────────────────────────────────┤
│                                     │
│  data_: shared_ptr<Mat>            │
│    └─> 存储张量实际数据              │
│         └─> 使用 Mat 抽象层         │
│              └─> 支持多后端          │
│                                     │
│  grad_: shared_ptr<Mat>            │
│    └─> 存储梯度数据                  │
│         └─> 初始为 nullptr          │
│              └─> backward() 时创建  │
│                                     │
│  creator_: FunctionPtr             │
│    └─> 记录创建者 Operator          │
│         └─> 用于构建计算图           │
│                                     │
│  generation_: int                  │
│    └─> 记录计算图代数                │
│         └─> 用于拓扑排序             │
│                                     │
└─────────────────────────────────────┘
```

**数据成员关系图：**

```
TensorImpl
  │
  ├─> data_ ──> Mat ──> Storage ──> 实际数据
  │
  ├─> grad_ ──> Mat ──> Storage ──> 梯度数据
  │
  └─> creator_ ──> Operator ──> 计算图连接
```

**方法职责划分：**

```
┌─────────────────────────────────────┐

│     └─> 反向传播入口                 │
│         └─> 自动构建计算图           │

```

### 3.3 Mat 抽象层

Mat 是矩阵计算的抽象接口层，隔离具体后端实现，支持多后端切换。

#### 3.3.1 Mat 接口设计

**核心接口：**
```cpp
class Mat {
public:
    // 基本操作
    virtual std::unique_ptr<Mat> clone() const = 0;
    virtual Shape shape() const = 0;
    virtual DataType dtype() const = 0;
    virtual Device device() const = 0;
    
    // 算术运算
    virtual std::unique_ptr<Mat> add(const Mat& other) const = 0;
    virtual std::unique_ptr<Mat> mul(const Mat& other) const = 0;
    virtual std::unique_ptr<Mat> matmul(const Mat& other) const = 0;
    
    // 形状操作
    virtual std::unique_ptr<Mat> reshape(const Shape& shape) const = 0;
    virtual std::unique_ptr<Mat> transpose() const = 0;
    
    // 原地操作（性能优化）
    virtual void add_inplace(const Mat& other) = 0;
    
    // 数据访问
    virtual void* data_ptr() = 0;
};
```

**设计特点：**
- **纯虚函数**：强制子类实现所有接口
- **返回 unique_ptr**：明确所有权转移
- **多态支持**：运行时选择具体实现

#### 3.3.2 后端隔离策略

**隔离方式：**
1. **接口与实现分离**
   - Mat 是抽象基类，定义接口
   - OriginMat、TorchMat 是具体实现

2. **工厂方法模式**
   ```cpp
   // 通过类型别名选择后端
   using Mat_t = OriginMat;  // 或 TorchMat
   
   // 工厂方法
   auto mat = Mat_t::from_scalar(1.0, shape, options);
   ```

3. **编译时选择**
   - 通过 `MAT_BACKEND` 宏在编译时选择
   - 不同后端编译成不同的库

**好处：**
- 算子层不依赖具体后端
- 新增后端只需实现 Mat 接口
- 用户代码无需修改

#### 3.3.3 多态机制的使用

**运行时多态：**
```cpp
std::shared_ptr<Mat> data_;  // 可以指向 OriginMat 或 TorchMat

// 调用时自动选择正确的实现
auto result = data_->add(*other_data_);  // 多态调用
```

**类型擦除：**
- 通过基类指针隐藏具体类型
- TensorImpl 不需要知道具体后端类型
- 运行时根据实际类型调用对应方法

### 3.4 Storage 层：数据拥有者

Storage 是数据的真正拥有者，管理原始内存，可以被多个 Mat 共享。

#### 3.4.1 Storage 的职责

**核心职责：**
1. **内存管理**
   - 分配和释放内存
   - 管理内存生命周期

2. **设备管理**
   - 支持 CPU 和 CUDA
   - 管理设备间的数据迁移

3. **引用计数**
   - 支持多个 Mat 共享同一个 Storage
   - 自动释放不再使用的内存

#### 3.4.2 内存管理（malloc/free）

**实现方式：**
```cpp
class Storage {
private:
    void* data_;              // 原始数据指针
    size_t size_;             // 大小（字节）
    DeviceType device_type_;  // 设备类型
    std::unique_ptr<Allocator> allocator_;  // 分配器
    
public:
    Storage(size_t size, DeviceType device_type, int device_index = 0) {
        allocator_ = AllocatorFactory::create_allocator(device_type, device_index);
        data_ = allocator_->allocate(size);
    }
    
    ~Storage() {
        if (data_ != nullptr) {
            allocator_->deallocate(data_);
        }
    }
};
```

**分配器模式：**
- 使用 Allocator 接口抽象内存分配
- 支持不同的分配策略（CPU、CUDA）
- 便于扩展和测试

#### 3.4.3 多 Mat 共享机制

**共享方式：**
```cpp
// Storage 通过 shared_ptr 管理
std::shared_ptr<Storage> storage_;

// 多个 Mat 可以共享同一个 Storage
Mat1::storage_ ──┐
                 ├──> shared_ptr<Storage>
Mat2::storage_ ──┘
```

**引用计数：**
- 使用 `shared_ptr` 自动管理引用计数
- 当最后一个 Mat 释放时，Storage 自动释放
- 线程安全（使用原子操作）

#### 3.4.4 引用计数管理

**实现细节：**
```cpp
class Storage {
private:
    std::atomic<int> ref_count_;  // 原子引用计数
    
public:
    void add_ref() {
        ref_count_.fetch_add(1);
    }
    
    void release() {
        int count = ref_count_.fetch_sub(1);
        if (count == 1) {
            delete this;  // 最后一个引用，释放内存
        }
    }
};
```

**注意：**
- 虽然 Storage 有自己的引用计数，但主要依赖 `shared_ptr`
- `shared_ptr` 已经提供了线程安全的引用计数
- Storage 的引用计数主要用于调试和统计

## 第4章：内存所有权设计

### 4.1 完整内存层次结构

OriginDL 的内存管理采用多层设计，每一层都有明确的职责：

```
Tensor (值语义包装)
    └─> TensorImplPtr (shared_ptr<TensorImpl>)
          └─> data_: shared_ptr<Mat>
                └─> OriginMat::storage_: shared_ptr<Storage>
                      └─> Storage::data_: void* (真正的数据)

          └─> grad_: shared_ptr<Mat>
                └─> OriginMat::storage_: shared_ptr<Storage>
                      └─> Storage::data_: void* (真正的数据)
```

**层次说明：**

1. **Tensor 层**
   - 值语义包装
   - 只包含一个 `shared_ptr<TensorImpl>`
   - 拷贝时共享 TensorImpl

2. **TensorImpl 层**
   - 管理 data_ 和 grad_
   - 管理计算图信息
   - 拷贝时深拷贝 data_ 和 grad_

3. **Mat 层**
   - 抽象接口，隐藏后端
   - 可以拥有 Storage 或作为视图
   - 拷贝时调用 clone()

4. **Storage 层**
   - 真正的数据拥有者
   - 管理原始内存
   - 可以被多个 Mat 共享

---

### 4.2 智能指针使用策略

OriginDL 使用三种智能指针，每种都有明确的用途：

#### 4.2.1 shared_ptr：数据共享

**使用场景：**
- `TensorImplPtr`：Tensor 共享 TensorImpl
- `data_`、`grad_`：TensorImpl 共享 Mat
- `storage_`：Mat 共享 Storage

**示例：**
```cpp
class TensorImpl {
    std::shared_ptr<Mat> data_;   // 多个 TensorImpl 可以共享
    std::shared_ptr<Mat> grad_;   // 多个 TensorImpl 可以共享
};

class OriginMat {
    std::shared_ptr<Storage> storage_;  // 多个 Mat 可以共享
};
```

**优势：**
- 自动内存管理
- 线程安全的引用计数
- 支持共享，避免不必要的拷贝

#### 4.2.2 unique_ptr：所有权转移

**使用场景：**
- 算子返回 Mat：`std::unique_ptr<Mat> forward(...)`
- 临时对象：函数内部创建的临时 Mat

**示例：**
```cpp
// 算子返回 unique_ptr
std::unique_ptr<Mat> Add::forward(const Mat& a, const Mat& b) {
    auto result = a.add(b);  // 返回 unique_ptr<Mat>
    return result;  // 转移所有权
}

// 转换为 shared_ptr
TensorImpl(std::unique_ptr<Mat> data) 
    : data_(std::shared_ptr<Mat>(std::move(data))) {}
```

**优势：**
- 明确所有权转移
- 避免不必要的拷贝
- 编译时检查所有权

#### 4.2.3 weak_ptr：避免循环引用

**使用场景：**
- `Operator::outputs_`：避免 Operator → Tensor → creator_ → Operator 的循环引用

**示例：**
```cpp
class Operator {
    std::vector<std::weak_ptr<TensorImpl>> outputs_;  // 使用 weak_ptr
};

// backward() 时转换为 shared_ptr
for (const auto& weak_output : f->outputs_) {
    auto output_impl = weak_output.lock();  // 转换为 shared_ptr
    if (output_impl) {
        // 使用 output_impl
    }
}
```

**优势：**
- 打破循环引用
- 自动检测对象是否已释放
- 不影响对象的生命周期

---

### 4.3 Storage 与 Mat 的关系

#### 4.3.1 Storage：数据拥有者

**职责：**
- 拥有真正的数据内存
- 管理内存的分配和释放
- 可以被多个 Mat 共享

**特点：**
- 不可拷贝（禁用拷贝构造函数）
- 只能移动（支持移动语义）
- 通过 shared_ptr 管理生命周期

#### 4.3.2 Mat：数据拥有者或视图

**两种模式：**

1. **数据拥有者模式**
   ```cpp
   // Mat 拥有 Storage
   OriginMat::storage_ = std::make_shared<Storage>(size, device);
   ```

2. **视图模式（View）**
   ```cpp
   // Mat 作为视图，共享 Storage
   auto view = mat.reshape(new_shape);  // 创建视图，共享 Storage
   ```

**视图的优势：**
- 零拷贝：reshape 连续张量时创建视图
- 内存效率：多个视图共享同一个 Storage
- 性能：避免不必要的数据拷贝

#### 4.3.3 视图机制：reshape/transpose 创建视图，共享 Storage

**实现策略：**

1. **连续张量：创建视图**
   ```cpp
   if (is_contiguous()) {
       // 创建视图，共享 Storage
       return create_view(new_shape);
   }
   ```

2. **非连续张量：创建新 Storage**
   ```cpp
   else {
       // 需要拷贝数据
       auto new_storage = create_new_storage();
       copy_data(new_storage);
       return new_mat;
   }
   ```

**判断连续性的标准：**
- 内存布局是否连续
- 维度顺序是否与内存顺序一致
- 是否有 stride 跳跃

### 4.4 拷贝语义设计

#### 4.4.1 Tensor 拷贝：浅拷贝（共享 TensorImpl）

**实现：**
```cpp
Tensor::Tensor(const Tensor &other) : impl_(other.impl_) {}
```

**行为：**
- 拷贝后，两个 Tensor 共享同一个 TensorImpl
- 修改一个 Tensor 的数据会影响另一个（如果直接操作底层数据）
- 但大部分操作返回新 Tensor，不会影响原对象

**示例：**
```cpp
auto x = Tensor::zeros({2, 3});
auto y = x;        // 浅拷贝，共享 TensorImpl
// x 和 y 指向同一个 TensorImpl
```

#### 4.4.2 TensorImpl 拷贝：深拷贝（clone data_ 和 grad_）

**实现：**
```cpp
TensorImpl::TensorImpl(const TensorImpl &other)
    : data_(other.data_ ? std::shared_ptr<Mat>(other.data_->clone()) : nullptr),
      grad_(other.grad_ ? std::shared_ptr<Mat>(other.grad_->clone()) : nullptr),
      creator_(other.creator_),
      generation_(other.generation_)
{}
```

**行为：**
- 深拷贝 data_ 和 grad_（调用 Mat::clone()）
- 共享 creator_（计算图信息）
- 拷贝 generation_

**为什么需要深拷贝？**
- 保证值语义：拷贝后独立
- 避免意外的数据共享
- 符合用户期望

#### 4.4.3 Mat 拷贝：clone() 方法

**实现：**
```cpp
virtual std::unique_ptr<Mat> clone() const = 0;
```

**行为：**
- 创建新的 Mat 对象
- 如果 Mat 拥有 Storage，创建新的 Storage 并拷贝数据
- 如果 Mat 是视图，创建新的视图（共享 Storage）

#### 4.4.4 何时需要深拷贝，何时可以共享

**需要深拷贝的场景：**
- TensorImpl 拷贝：保证值语义
- 用户显式调用 clone()
- 需要独立的数据副本

**可以共享的场景：**
- Tensor 拷贝：值语义，但底层共享
- 视图操作：reshape 连续张量时创建视图
- 梯度共享：多个路径的梯度可以共享（但需要累加）

**设计原则：**
- **默认共享**：提高性能，减少内存使用
- **按需拷贝**：只在必要时才深拷贝
- **用户可控**：提供 clone() 方法让用户显式控制

### 4.5 内存优化策略

#### 4.5.1 零拷贝视图（连续张量的 reshape）

**优化策略：**
```cpp
TensorImpl TensorImpl::reshape(const Shape &shape) const {
    if (data_->is_contiguous()) {
        // 创建视图，零拷贝
        auto view = data_->view(shape);
        return TensorImpl(std::move(view));
    } else {
        // 非连续，需要拷贝
        auto new_data = data_->clone();
        new_data->reshape(shape);
        return TensorImpl(std::move(new_data));
    }
}
```

**优势：**
- **性能**：避免数据拷贝，O(1) 时间复杂度
- **内存**：多个视图共享同一个 Storage
- **灵活性**：支持多种形状变换

#### 4.5.2 原地操作（add_inplace 用于梯度累加）

**优化策略：**
```cpp
// 梯度累加时使用原地操作
if (!x.impl_->grad_) {
    // 梯度为空，直接共享
    x.impl_->grad_ = gx.impl_->data_;
} else {
    // 梯度不为空，原地累加（避免创建新对象）
    x.impl_->grad_->add_inplace(*gx.impl_->data_);
}
```

**优势：**
- **性能**：避免创建新的 Mat 对象
- **内存**：复用现有内存，不分配新内存
- **效率**：直接修改现有数据

#### 4.5.3 内存复用（避免不必要的内存分配）

**优化策略：**
1. **对象池**：复用 TensorImpl 对象（可选）
2. **延迟分配**：梯度只在需要时分配
3. **智能共享**：尽可能共享 Storage

**示例：**
```cpp
// 梯度延迟分配
void TensorImpl::backward() {
    if (!grad_) {
        // 只在需要时分配梯度内存
        grad_ = std::shared_ptr<Mat>(Mat_t::zeros(shape(), options()));
    }
    // ...
}
```

## 第5章：Tensor 构造函数与工厂方法

### 5.1 构造函数设计

Tensor 提供了多种构造函数，支持不同的创建方式。

#### 5.1.1 从 unique_ptr<Mat> 构造（算子返回）

**用途：** 算子计算后返回 Mat，需要转换为 Tensor

**实现：**
```cpp
Tensor::Tensor(std::unique_ptr<Mat> mat) 
    : impl_(std::make_shared<TensorImpl>(std::move(mat))) {}
```

**调用路径：**
```
Operator::forward()
  ↓ 返回 unique_ptr<Mat>
convert_mat_to_tensor(std::move(result))
  ↓ 调用 Tensor(std::unique_ptr<Mat>)
TensorImpl(std::unique_ptr<Mat>)
  ↓ unique_ptr -> shared_ptr 转换
```

**设计考虑：**
- 算子返回 `unique_ptr`，明确所有权转移
- Tensor 构造函数接受 `unique_ptr`，转换为 `shared_ptr`
- 一次所有权转移，避免不必要的拷贝

#### 5.1.2 从 shared_ptr<Mat> 构造（内部共享）

**用途：** 内部需要共享 Mat 时使用

**实现：**
```cpp
TensorImpl::TensorImpl(std::shared_ptr<Mat> data) 
    : data_(data), grad_(nullptr), creator_(nullptr), generation_(0) {}
```

**使用场景：**
- 视图操作：创建视图时共享 Mat
- 内部优化：避免不必要的拷贝

#### 5.1.3 从标量构造（from_scalar）

**用途：** 创建填充标量值的张量

**实现：**
```cpp
Tensor::Tensor(const Scalar& value, const Shape& shape, const TensorOptions& options) {
    auto impl = TensorImpl::from_scalar(value, shape, options);
    impl_ = std::make_shared<TensorImpl>(std::move(impl));
}

TensorImpl TensorImpl::from_scalar(const Scalar& scalar, const Shape& shape, const TensorOptions& options) {
    auto mat = Mat_t::from_scalar(scalar, shape, options);
    return TensorImpl(std::move(mat));
}
```

**调用链：**
```
Tensor(value, shape, options)
  ↓
TensorImpl::from_scalar()
  ↓
Mat_t::from_scalar()  // 后端工厂方法
  ↓
创建 Mat 并填充标量值
```

#### 5.1.4 从内存构造（from_memory）

**用途：** 从已有内存创建张量（零拷贝或拷贝）

**实现：**
```cpp
Tensor Tensor::from_blob(void* data, const Shape& shape, const TensorOptions& options) {
    Tensor result;
    result.from_memory(data, options.dtype(), shape, options);
    return result;
}

void Tensor::from_memory(const void* data, DataType dtype, const Shape& shape, const TensorOptions& options) {
    auto impl = TensorImpl::from_memory(data, dtype, shape, options);
    impl_ = std::make_shared<TensorImpl>(std::move(impl));
}
```

**使用场景：**
- 从 C 数组创建张量
- 从其他库的数据创建张量
- 与外部数据交互

### 5.3 类型转换

#### 5.3.1 to() 方法（dtype、device 转换）

**接口：**
```cpp
TensorImpl TensorImpl::to(const TensorOptions& options) const;
Tensor Tensor::to(const TensorOptions& options) const;
```

**功能：**
- 转换数据类型（float32 → float64）
- 转换设备（CPU → CUDA）
- 返回新的 Tensor（不修改原对象）

**实现：**
```cpp
TensorImpl TensorImpl::to(const TensorOptions& options) const {
    // 检查是否需要转换
    if (data_->dtype() == options.dtype() && 
        data_->device() == options.device()) {
        return *this;  // 无需转换
    }
    
    // 创建新的 Mat（转换类型或设备）
    auto new_mat = data_->to(options);
    return TensorImpl(std::move(new_mat));
}
```

#### 5.3.2 类型提升规则

**规则：**
- 低精度 → 高精度：自动提升
- 高精度 → 低精度：需要显式转换（可能丢失精度）
- 相同类型：无需转换

**示例：**
```cpp
auto x = Tensor::zeros({2, 3}, dtype("float32"));
auto y = x.to(dtype("float64"));  // float32 → float64
```

#### 5.3.3 设备迁移机制

**CPU ↔ CUDA 迁移：**
```cpp
// CPU → CUDA
auto cpu_tensor = Tensor::zeros({2, 3}, device("cpu"));
auto cuda_tensor = cpu_tensor.to(device("cuda:0"));

// CUDA → CPU
auto back_to_cpu = cuda_tensor.to(device("cpu"));
```

**实现细节：**
- 需要拷贝数据（不同设备）
- 异步传输（CUDA）
- 自动同步（确保数据一致性）
