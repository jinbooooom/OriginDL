# 第三部分：算子系统架构

## 第6章：Operator 基类设计

### 6.1 算子系统整体架构

算子系统是 OriginDL 的核心计算单元，负责执行前向传播和反向传播。系统采用统一的 Operator 基类设计，支持多输入多输出，并通过计算图管理算子之间的关系。

#### 6.1.1 算子系统架构图

**整体架构：**

```
┌─────────────────────────────────────────────────────────┐
│  用户代码层                                             │
│  Tensor y = x0 + x1;                                   │
└─────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│  Tensor 运算符重载层                                     │
│  operator+(x0, x1)                                      │
│    └─> 创建算子实例                                      │
└─────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│  Operator 基类层                                        │
│  ┌──────────────────────────────────────┐               │
│  │ class Operator                      │               │
│  │   +forward() = 0                    │               │
│  │   +backward() = 0                   │               │
│  │   +inputs_                          │               │
│  │   +outputs_ (weak_ptr)              │               │
│  │   +generation_                       │               │
│  └──────────────────────────────────────┘               │
│              ▲                                          │
│              │ 继承                                     │
│    ┌─────────┴─────────┐                              │
│    │                    │                              │
│ ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐              │
│ │ Add  │  │ Mul  │  │ ReLU │  │Linear│  ...          │
│ └──────┘  └──────┘  └──────┘  └──────┘              │
└─────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│  Mat 接口层（多态调用）                                 │
│  mat(x0) + mat(x1)                                     │
│    └─> OriginMat::add() 或 TorchMat::add()             │
└─────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│  后端计算层                                             │
│  Storage → void* (实际计算)                             │
└─────────────────────────────────────────────────────────┘
```

**类图（继承关系）：**

```
┌─────────────────────────────────────┐
│         Operator                    │  (抽象基类)
│─────────────────────────────────────│
│ +inputs_: vector<Tensor>            │
│ +outputs_: vector<weak_ptr<TensorImpl>>│
│ +generation_: int                   │
│                                     │
│ +forward(inputs) = 0               │
│ +backward(grad_outputs) = 0         │
│ +operator()(inputs)                 │
│ +setup_computation_graph()          │
└─────────────────────────────────────┘
              ▲
              │ 继承
    ┌─────────┼─────────┬─────────┐
    │         │         │         │
┌──────┐ ┌──────┐ ┌──────┐ ┌──────────┐
│ Add  │ │ Mul  │ │ ReLU │ │ LinearOp │
│──────│ │──────│ │──────│ │──────────│
│+forward│ │+forward│ │+forward│ │+forward  │
│+backward│ │+backward│ │+backward│ │+backward │
└──────┘ └──────┘ └──────┘ └──────────┘
```

**计算图连接关系：**

```
前向传播时构建计算图：

x0 (generation=0) ──┐
                   ├─> [Add Operator]
x1 (generation=0) ──┘
                     │
                     ▼
                   y (generation=1)
                     │
                     ├─> creator_ = Add
                     └─> Add::outputs_ = {weak_ptr(y.impl_)}

反向传播时遍历计算图：

y.backward()
  ↓
TensorImpl::backward()
  ↓
拓扑排序: [Add]
  ↓
Add::backward({gy})
  ↓
返回 {gx0, gx1}
  ↓
梯度累积到 x0.grad_ 和 x1.grad_
```

#### 6.1.2 算子调用流程原理

**完整调用链：**

```
┌─────────────────────────────────────────────────────┐
│ 1. 用户代码                                         │
│    Tensor y = x0 + x1;                            │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│ 2. Tensor 运算符重载                                 │
│    operator+(x0, x1)                               │
│      └─> 创建 Add 算子实例                          │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│ 3. Operator::operator()                             │
│    auto outputs = forward(inputs);                  │
│    setup_computation_graph(inputs, outputs);        │
│    return outputs;                                  │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│ 4. Add::forward()                                   │
│    auto result = mat(xs[0]) + mat(xs[1]);          │
│      └─> 调用 Mat 接口（多态）                       │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│ 5. Mat 接口层（多态调用）                           │
│    OriginMat::operator+() 或 TorchMat::operator+()  │
│      └─> 返回 unique_ptr<Mat>                       │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│ 6. 转换为 Tensor                                    │
│    convert_mat_to_tensor(std::move(result))         │
│      └─> Tensor(std::unique_ptr<Mat>)               │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│ 7. 设置计算图信息                                    │
│    output.set_creator(shared_from_this());          │
│    output.generation_ = input.generation_ + 1;      │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│ 8. 返回结果                                          │
│    return Tensor y;                                │
└─────────────────────────────────────────────────────┘
```

**关键转换点：**

1. **Tensor → Mat**：通过 `impl_->data_` 获取 Mat 引用
2. **Mat → unique_ptr<Mat>**：算子返回计算结果
3. **unique_ptr → shared_ptr**：在 TensorImpl 构造函数中转换
4. **设置 creator_**：使用 `shared_from_this()` 获取 Operator 的 shared_ptr

### 6.2 Operator 接口设计

Operator 是所有算子的基类，定义了算子的统一接口和行为。

#### 6.2.1 enable_shared_from_this 的使用

**设计原理：**

`enable_shared_from_this` 允许对象在内部安全地获取自身的 `shared_ptr`。这在设置 Tensor 的 `creator_` 时是必需的，因为 `creator_` 需要 `shared_ptr<Operator>` 而不是原始指针。

**为什么需要：**

```
问题场景：
  Operator::operator() 需要设置 output.creator_ = this
  但 creator_ 的类型是 shared_ptr<Operator>
  不能直接使用 this（原始指针）

解决方案：
  使用 enable_shared_from_this
  通过 shared_from_this() 获取自身的 shared_ptr
```

**实现：**

```cpp
class Operator : public std::enable_shared_from_this<Operator> {
public:
    std::vector<Tensor> operator()(const std::vector<Tensor> &inputs) {
        auto outputs = this->forward(inputs);
        
        // 设置 creator，需要 shared_ptr<Operator>
        for (auto &output : outputs) {
            output.set_creator(shared_from_this());  // 获取自身的 shared_ptr
        }
        
        return outputs;
    }
};
```

**内存关系图：**

```
Operator (通过 shared_ptr 管理)
  │
  ├─> shared_from_this() 返回 shared_ptr<Operator>
  │     │
  │     └─> 设置到 Tensor 的 creator_
  │           │
  │           ▼
  │     TensorImpl::creator_ = shared_ptr<Operator>
  │
  └─> 计算图连接：
        Operator
          ├─> inputs_ (vector<Tensor>)
          └─> outputs_ (vector<weak_ptr<TensorImpl>>)
```

**使用场景：**
- 设置 Tensor 的 creator_
- 在 backward() 中访问 Operator
- 管理 Operator 的生命周期

### 6.2 算子调用流程

#### 6.2.1 完整调用路径

**用户代码到后端计算的完整流程：**

```
用户代码: Tensor y = x0 + x1;
  ↓
operator+(x0, x1)  // Tensor 运算符重载
  ↓
Add::forward({x0, x1})  // 创建 Add 算子，调用 forward
  ↓
mat(x0) + mat(x1)  // 获取 Mat 引用，调用 Mat::add()
  ↓
OriginMat::add()  // 后端实现，返回 unique_ptr<Mat>
  ↓
convert_mat_to_tensor(std::move(result))  // unique_ptr<Mat> -> Tensor
  ↓
Tensor(std::unique_ptr<Mat>)  // Tensor 构造函数
  ↓
TensorImpl(std::unique_ptr<Mat>)  // unique_ptr -> shared_ptr 转换
  ↓
设置 creator_ 和计算图信息
  ↓
返回 Tensor y
```

#### 6.2.2 关键转换点

**类型转换架构：**

```
转换流程：

1. Tensor → Mat（获取后端引用）
   ┌──────────┐
   │  Tensor  │
   └──────────┘
        │ impl_->data_
        ▼
   ┌──────────┐
   │   Mat    │  ← 获取引用
   └──────────┘

2. Mat → Tensor（转换计算结果）
   ┌──────────┐
   │unique_ptr│
   │   Mat    │
   └──────────┘
        │ std::move
        ▼
   ┌──────────┐
   │  Tensor  │
   └──────────┘

3. unique_ptr → shared_ptr（所有权转换）
   ┌──────────┐
   │unique_ptr│
   │   Mat    │
   └──────────┘
        │ 在 TensorImpl 构造函数中
        ▼
   ┌──────────┐
   │shared_ptr│
   │   Mat    │  ← 开始共享管理
   └──────────┘
```

#### 6.2.3 示例：Add 算子实现

**前向传播：**
```cpp
std::vector<Tensor> Add::forward(const std::vector<Tensor> &xs) {
    // 1. 保存输入形状（用于 backward）
    shape0_ = xs[0].shape();
    shape1_ = xs[1].shape();
    
    // 2. 调用后端计算
    auto result = mat(xs[0]).add(mat(xs[1]));  // 返回 unique_ptr<Mat>
    
    // 3. 转换为 Tensor
    return {convert_mat_to_tensor(std::move(result))};
}
```

**反向传播：**
```cpp
std::vector<Tensor> Add::backward(const std::vector<Tensor> &gys) {
    auto &gy = gys[0];
    
    // 梯度计算：gx0 = gy, gx1 = gy
    // 但需要考虑广播
    auto gx0 = broadcast_to(gy, shape0_);
    auto gx1 = broadcast_to(gy, shape1_);
    
    return {gx0, gx1};
}
```

### 6.3 循环引用问题与解决方案

#### 6.3.1 问题：循环引用

**循环引用链：**
```
Operator → outputs_ (shared_ptr<Tensor>) 
    → Tensor → impl_ (shared_ptr<TensorImpl>)
        → creator_ (shared_ptr<Operator>)
            → Operator (循环！)
```

**问题：**
- 使用 `shared_ptr` 会导致循环引用
- 对象无法自动释放，造成内存泄漏

#### 6.3.2 解决方案：outputs_ 使用 weak_ptr

**设计：**
```cpp
class Operator {
public:
    // 使用 weak_ptr 避免循环引用
    std::vector<std::weak_ptr<TensorImpl>> outputs_;
};
```

**为什么使用 weak_ptr：**
- **不拥有对象**：weak_ptr 不增加引用计数
- **打破循环**：Operator 不持有 Tensor 的强引用
- **安全访问**：可以通过 lock() 转换为 shared_ptr

#### 6.3.3 backward() 时 weak_ptr 转 shared_ptr

**实现：**
```cpp
void TensorImpl::backward() {
    // ...
    for (const auto &weak_output : f->outputs_) {
        // 将 weak_ptr 转换为 shared_ptr
        auto output_impl = weak_output.lock();
        if (!output_impl) {
            // weak_ptr 失效，说明 Tensor 已释放
            continue;  // 跳过
        }
        
        // 使用 output_impl
        Tensor output_tensor(output_impl);
        gys.push_back(output_tensor.grad());
    }
    // ...
}
```

**生命周期管理：**
- **forward() 时**：保存 weak_ptr，不持有强引用
- **backward() 时**：临时转换为 shared_ptr，使用完后释放
- **Tensor 释放**：如果 Tensor 已释放，weak_ptr.lock() 返回 nullptr

#### 6.3.4 生命周期管理策略

**策略 1：弱引用观察**
- Operator 不拥有输出 Tensor
- 只观察输出 Tensor 是否存在
- 如果 Tensor 已释放，说明用户不再需要，跳过即可

**策略 2：临时强引用**
- backward() 时临时转换为 shared_ptr
- 确保在 backward() 期间 Tensor 有效
- backward() 结束后自动释放

**策略 3：自动清理**
- backward() 完成后清理 inputs_ 和 outputs_
- 断开计算图连接
- 释放不再需要的内存

## 第7章：常见算子实现

### 7.1 算术算子

算术算子是深度学习框架中最基础的算子，包括加法、减法、乘法、除法等。

#### 7.1.1 Add：前向与反向实现

**前向传播：**
```cpp
std::vector<Tensor> Add::forward(const std::vector<Tensor> &xs) {
    // 1. 验证输入
    if (xs.size() != 2) {
        THROW_RUNTIME_ERROR("Add requires 2 inputs");
    }
    
    // 2. 保存输入形状（用于 backward）
    shape0_ = xs[0].shape();
    shape1_ = xs[1].shape();
    
    // 3. 类型提升（如果需要）
    if (TypePromotion::needs_promotion(xs)) {
        auto promoted = TypePromotion::promote_tensors(xs);
        auto result = mat(promoted[0]) + mat(promoted[1]);
        return {convert_mat_to_tensor(std::move(result))};
    }
    
    // 4. 直接计算
    auto result = mat(xs[0]) + mat(xs[1]);
    return {convert_mat_to_tensor(std::move(result))};
}
```

**反向传播：**
```cpp
std::vector<Tensor> Add::backward(const std::vector<Tensor> &gys) {
    auto &gy = gys[0];
    
    // 梯度：gx0 = gy, gx1 = gy
    // 但需要考虑广播：如果输入形状不同，需要 sum_to
    auto gx0 = gy;
    auto gx1 = gy;
    
    if (shape0_ != shape1_) {
        // 广播回原始形状
        if (gx0.shape() != shape0_) {
            gx0 = sum_to(gx0, shape0_);
        }
        if (gx1.shape() != shape1_) {
            gx1 = sum_to(gx1, shape1_);
        }
    }
    
    return {gx0, gx1};
}
```

**关键点：**
- **类型提升**：支持不同数据类型的加法
- **广播处理**：支持不同形状的加法
- **梯度广播**：反向传播时需要将梯度广播回原始形状

#### 7.1.2 Sub：前向与反向实现

**前向传播：**
```cpp
std::vector<Tensor> Sub::forward(const std::vector<Tensor> &xs) {
    // 类似 Add，但使用减法
    auto result = mat(xs[0]) - mat(xs[1]);
    return {convert_mat_to_tensor(std::move(result))};
}
```

**反向传播：**
```cpp
std::vector<Tensor> Sub::backward(const std::vector<Tensor> &gys) {
    // 梯度：gx0 = gy, gx1 = -gy
    auto gx0 = gys[0];
    auto gx1 = neg(gys[0]);  // 取负
    return {gx0, gx1};
}
```

#### 7.1.3 Mul：前向与反向实现（元素级乘法）

**前向传播：**
```cpp
std::vector<Tensor> Mul::forward(const std::vector<Tensor> &xs) {
    // 元素级乘法（不是矩阵乘法）
    auto result = mat(xs[0]) * mat(xs[1]);
    return {convert_mat_to_tensor(std::move(result))};
}
```

**反向传播：**
```cpp
std::vector<Tensor> Mul::backward(const std::vector<Tensor> &gys) {
    // 梯度：gx0 = gy * x1, gx1 = gy * x0
    auto &x0 = inputs_[0];
    auto &x1 = inputs_[1];
    auto &gy = gys[0];
    
    auto gx0 = gy * x1;
    auto gx1 = gy * x0;
    
    return {gx0, gx1};
}
```

**注意：**
- Mul 是元素级乘法，不是矩阵乘法
- 矩阵乘法使用 MatMul 算子

#### 7.1.4 Div：前向与反向实现

**前向传播：**
```cpp
std::vector<Tensor> Div::forward(const std::vector<Tensor> &xs) {
    auto result = mat(xs[0]) / mat(xs[1]);
    return {convert_mat_to_tensor(std::move(result))};
}
```

**反向传播：**
```cpp
std::vector<Tensor> Div::backward(const std::vector<Tensor> &gys) {
    // 梯度：gx0 = gy / x1, gx1 = -gy * x0 / (x1^2)
    auto &x0 = inputs_[0];
    auto &x1 = inputs_[1];
    auto &gy = gys[0];
    
    auto gx0 = gy / x1;
    auto gx1 = neg(gy * x0 / square(x1));
    
    return {gx0, gx1};
}
```

#### 7.4.4 Softmax

**前向传播：**
```cpp
std::vector<Tensor> Softmax::forward(const std::vector<Tensor> &xs) {
    // softmax(x) = exp(x - max(x)) / sum(exp(x - max(x)))
    // 数值稳定性：先减去最大值
    auto x = xs[0];
    auto x_max = max(x, axis_);
    auto x_shifted = x - x_max;
    auto exp_x = exp(x_shifted);
    auto sum_exp = sum(exp_x, axis_);
    auto y = exp_x / sum_exp;
    
    return {y};
}
```

**反向传播：**
```cpp
std::vector<Tensor> Softmax::backward(const std::vector<Tensor> &gys) {
    // 梯度计算较复杂，需要保存中间结果
    // gx = y * (gy - sum(gy * y))
    auto &y = saved_output_;  // 前向传播保存的结果
    auto &gy = gys[0];
    
    auto gy_y = gy * y;
    auto sum_gy_y = sum(gy_y, axis_);
    auto gx = y * (gy - sum_gy_y);
    
    return {gx};
}
```

**特点：**
- 数值稳定性：减去最大值避免溢出
- 需要保存中间结果用于反向传播

```cpp
    // gbias = sum(gy, dim=0)
    auto &gy = gys[0];
    auto &x = inputs_[0];
    auto &weight = inputs_[1];
```
